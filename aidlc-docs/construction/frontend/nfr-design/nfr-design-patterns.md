# NFR 설계 패턴 (NFR Design Patterns) - UoW-2 Frontend

## 실시간 패턴 (Realtime)
- **SSE 구독 훅(useSse)**: EventSource를 감싸 컴포넌트 마운트 시 연결, 언마운트 시 해제. 이벤트 콜백으로 상태 업데이트.
- **자동 재연결**: onerror 시 지수 백오프(예: 1s→2s→4s, 상한 30s)로 재연결(FE-NFR-R1).
- **낙관적 갱신 최소화**: 상태 변경/삭제는 서버 응답 + SSE 이벤트로 화면 갱신(단일 진실 원천).

## 인증 패턴 (Auth)
- **라우트 가드**: 고객/관리자 영역 진입 시 localStorage 토큰 확인. 없으면 로그인/설정 화면으로 이동.
- **토큰 자동 첨부**: ApiClient가 저장된 토큰을 Authorization 헤더에 첨부.
- **401 처리**: 401 응답 시 토큰 제거 후 로그인 화면 이동.

## 상태 관리 패턴 (State)
- **로컬 우선(Local-first) 장바구니**: CartStore가 localStorage를 단일 소스로 사용, React state는 미러링.
- **컨텍스트 최소화**: 전역 상태 라이브러리 미도입. 필요한 곳에서 localStorage 모듈 직접 사용 + 컴포넌트 state.

## 성능 패턴
- **SSE 기반 푸시**: 폴링 대신 이벤트 푸시로 즉시 반영(FE-NFR-P1).
- **부분 갱신**: 대시보드는 테이블 카드 단위로 상태 갱신.

## 사용성 패턴
- **터치 타깃**: 버튼 최소 44x44px CSS 적용.
- **신규 주문 강조**: 신규 order_created 카드에 강조 클래스/애니메이션 후 일정 시간 뒤 해제.

## 에러 처리 패턴
- **에러 배너/메시지**: API 실패 시 사용자 메시지 표시, 주문 실패 시 장바구니 유지(FE-NFR-R2).
