## 3. 기능 요구사항 (Functional Requirements)

### 3.1 고객용 기능 (Customer Features)

#### FR-C1: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정: 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 최종 로그인 정보 로컬 저장(브라우저 localStorage)
- 1회 로그인 성공 후 저장된 정보로 자동 로그인

#### FR-C2: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시, 카테고리 간 빠른 이동
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지(URL)
- UI: 카드 형태 레이아웃, 터치 친화 버튼(최소 44x44px), 명확한 시각 계층

#### FR-C3: 장바구니 관리
- 메뉴 추가/삭제, 수량 증감, 총 금액 실시간 계산, 장바구니 비우기
- 클라이언트 측 로컬 저장(페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C4: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 성공 플로우: 주문 번호 표시 → 장바구니 자동 비우기 → 5초 후 메뉴 화면으로 자동 리다이렉트
- 실패 시: 에러 메시지 표시 + 장바구니 유지
- 주문 정보: 매장 식별 정보, 테이블 식별 정보, 메뉴 목록(메뉴명/수량/단가), 총 금액, 세션 ID

#### FR-C5: 주문 내역 조회 (현재 세션)
- 주문 시간 순 정렬, 주문별 상세(주문 번호, 시각, 메뉴/수량, 금액, 상태)
- **현재 테이블 세션 주문만 표시** (이전 세션 및 이용 완료 주문 제외)
- 주문 상태(대기중/준비중/완료) **SSE 실시간 업데이트** (Q10: 포함)
- 페이지네이션 또는 무한 스크롤

### 3.2 관리자용 기능 (Admin Features)

#### FR-A1: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, **16시간 세션 유지**, 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱 저장, 로그인 시도 제한

#### FR-A2: 실시간 주문 모니터링
- **SSE 기반** 주문 목록 실시간 업데이트 (2초 이내 표시)
- 그리드/대시보드 레이아웃: 테이블별 카드, 카드에 총 주문액 + 최신 주문 n개 미리보기
- 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경(대기중/준비중/완료), 신규 주문 시각적 강조(색상/애니메이션)
- 테이블별 필터링

#### FR-A3: 테이블 관리
1. **테이블 태블릿 초기 설정**: 테이블 번호/비밀번호 설정, 16시간 세션 생성, 자동 로그인 활성화
2. **주문 삭제(직권 수정)**: 삭제 버튼 → 확인 팝업 → 즉시 삭제 → 총 주문액 재계산 → 성공/실패 피드백
3. **테이블 세션 처리**: 세션 시작(첫 주문)/종료(이용 완료) 관리, 확인 팝업, 종료 시 주문 내역을 과거 이력으로 이동 + 현재 주문 목록/총액 0으로 리셋, 새 고객 시작 가능
4. **과거 주문 내역 조회**: "과거 내역" 버튼, 테이블별 과거 주문(시간 역순), 각 주문 정보(번호/시각/메뉴/총액/이용 완료 시각), 날짜 필터링, "닫기"로 복귀

#### FR-A4: 메뉴 관리
- 카테고리별 조회, 등록(메뉴명/가격/설명/카테고리/이미지 URL), 수정, 삭제, 노출 순서 조정
- 데이터 검증: 필수 필드 검증, 가격 범위 검증

### 3.3 데이터 모델 (개략)
- **Store** (매장): 매장 식별자, 매장명
- **AdminUser** (관리자): 사용자명, 비밀번호 해시, 매장 참조
- **Table** (테이블): 테이블 번호, 테이블 비밀번호 해시, 현재 세션 참조
- **TableSession** (테이블 세션): 세션 ID, 테이블 참조, 시작 시각, 종료 시각, 상태(활성/종료)
- **Category** (카테고리): 카테고리명, 노출 순서
- **MenuItem** (메뉴): 메뉴명, 가격, 설명, 카테고리 참조, 이미지 URL, 노출 순서
- **Order** (주문): 주문 번호, 테이블/세션 참조, 주문 시각, 상태(대기중/준비중/완료), 총 금액
- **OrderItem** (주문 항목): 주문 참조, 메뉴명(스냅샷), 단가(스냅샷), 수량
- **OrderHistory** (주문 이력): 세션 종료 시 이동된 과거 주문, 이용 완료 시각

## 4. 비기능 요구사항 (Non-Functional Requirements)

### NFR-1: 성능
- 관리자 주문 모니터링: 신규 주문 **2초 이내** 표시 (SSE)
- 고객 주문 생성 응답: 즉각적(체감 1초 내외)

### NFR-2: 실시간성
- 관리자 대시보드 및 고객 주문 내역 모두 **Server-Sent Events(SSE)** 기반 실시간 통신

### NFR-3: 인증/세션
- 관리자: JWT 기반, 16시간 세션, bcrypt 비밀번호 해싱, 로그인 시도 제한
- 고객 태블릿: 로컬 저장 정보로 자동 로그인
- (보안 확장 규칙은 미적용 — Q7. 단, 위 명시된 인증 요구사항은 기능 요구사항으로 구현)

### NFR-4: 사용성 (UI/UX)
- 터치 친화적 인터페이스(최소 44x44px 버튼), 카드 기반 레이아웃, 명확한 시각 계층
- 고객용/관리자용 별도 인터페이스

### NFR-5: 데이터 지속성
- SQLite에 매장/메뉴/주문/이력 영속 저장
- 장바구니는 클라이언트 로컬 저장(새로고침 유지)

### NFR-6: 테스트 품질 (PBT - Partial 적용)
- 속성 기반 테스트 프레임워크 선정 및 의존성 추가 (PBT-09) → Python: **Hypothesis**
- 직렬화/역직렬화, 파싱/포매팅 등 왕복 속성 테스트 (PBT-02)
- 불변식 속성 테스트: 총 주문액 계산, 세션 종료 후 리셋 등 비즈니스 불변식 (PBT-03)
- 도메인 특화 제너레이터 사용 (PBT-07)
- 셰이딩(shrinking) 및 시드 재현성 보장 (PBT-08)
- 적용 범위: 순수 함수 및 직렬화 왕복 중심 (Partial 모드)

### NFR-7: 배포/실행
- 로컬 개발 환경에서 단일 머신 실행 (백엔드 + 두 개의 React 프론트엔드 + SQLite)
- 시드 스크립트로 데모 데이터 준비

## 5. MVP 범위 (구현 대상)

**고객용**: 자동 로그인/세션, 메뉴 조회, 장바구니, 주문 생성(5초 후 리다이렉트), 주문 내역 조회(현재 세션, SSE 실시간)
**관리자용**: 매장 인증(16시간), 실시간 주문 모니터링(그리드, SSE), 테이블 관리(초기 설정/주문 삭제/이용 완료/과거 내역), 메뉴 관리

## 6. 제외 사항 (Out of Scope)

`requirements/constraints.md`에 명시된 모든 항목 제외:
- 실제 결제 처리, PG 연동, 영수증, 환불, 포인트/쿠폰
- OAuth/SNS 로그인, 2FA/OTP
- 이미지 리사이징/최적화, CMS, 광고
- 푸시/SMS/이메일/소리 알림
- 주방 전달, 주방 재고 관리
- 데이터 분석/대시보드, 매출 리포트, 재고 관리, 직원 관리, 예약, 리뷰, 다국어
- 배달 플랫폼/POS 연동, 소셜 공유, 지도/번역 API

## 7. 핵심 요구사항 요약

테이블오더 MVP는 **단일 매장** 대상의 디지털 주문 시스템으로, **FastAPI 백엔드 + SQLite + React 프론트엔드(고객용/관리자용)** 로 구성됩니다. 핵심은 (1) 고객의 무로그인 즉시 주문 경험, (2) 관리자의 SSE 기반 실시간 주문 모니터링, (3) 테이블 세션 라이프사이클(시작~이용 완료) 관리와 과거 이력 분리입니다. 보안 강화 확장은 적용하지 않으며, 속성 기반 테스트는 Partial 모드로 순수 함수/직렬화 중심으로 적용합니다.
