# 사용자 페르소나 (User Personas)

테이블오더 서비스의 주요 사용자 유형을 정의합니다.

---

## 페르소나 1: 매장 고객 (Customer)

**이름(가상)**: 김민수 (Customer)

**유형**: 매장에 방문하여 테이블에 비치된 태블릿으로 주문하는 일반 고객

### 특성 (Characteristics)
- 디지털 기기 사용에 익숙하나, 별도 학습 없이 즉시 사용 가능해야 함
- 테이블에 앉아 빠르게 메뉴를 보고 주문하기를 원함
- 로그인/회원가입 같은 절차를 원하지 않음 (태블릿은 이미 로그인된 상태)

### 목표 (Goals)
- 대기 없이 즉시 메뉴를 탐색하고 주문
- 주문한 내역과 진행 상태를 확인
- 추가 주문을 간편하게 수행

### 동기 (Motivations)
- 빠르고 편리한 주문 경험
- 직원을 호출하지 않고 스스로 주문 완료
- 주문이 제대로 접수되었는지에 대한 확신

### 페인포인트 (Pain Points)
- 복잡한 UI나 작은 버튼으로 인한 터치 오류
- 주문이 접수되었는지 불확실한 상황
- 이전 손님의 주문 내역이 섞여 보이는 혼란

### 관련 기능
- 자동 로그인/세션, 메뉴 조회, 장바구니, 주문 생성, 주문 내역 조회

## 페르소나 2: 매장 관리자/운영자 (Admin)

**이름(가상)**: 이지영 (Store Admin)

**유형**: 매장을 운영하며 들어오는 주문을 관리하고 메뉴/테이블을 설정하는 운영자

### 특성 (Characteristics)
- 매장 운영 중 여러 테이블의 주문을 동시에 모니터링해야 함
- 바쁜 상황에서 한눈에 주문 현황을 파악하기를 원함
- 초기 1회 설정(테이블 태블릿, 메뉴) 이후에는 주로 모니터링/상태 변경 작업 수행

### 목표 (Goals)
- 들어오는 주문을 실시간으로 빠짐없이 확인
- 테이블별 주문 현황과 총액을 한눈에 파악
- 주문 상태를 갱신하고, 잘못된 주문을 정정
- 테이블 이용 완료 처리 및 과거 내역 조회
- 메뉴를 자유롭게 등록/수정/삭제

### 동기 (Motivations)
- 인건비 절감 및 운영 효율 증대
- 주문 누락 방지 및 신속한 처리
- 정확한 매출/주문 관리

### 페인포인트 (Pain Points)
- 주문 알림 지연으로 인한 누락
- 테이블별 주문 상태 파악의 어려움
- 세션이 섞여 이전 고객 주문과 현재 고객 주문이 구분되지 않는 문제

### 관련 기능
- 매장 인증, 실시간 주문 모니터링, 테이블 관리(초기 설정/주문 삭제/이용 완료/과거 내역), 메뉴 관리
