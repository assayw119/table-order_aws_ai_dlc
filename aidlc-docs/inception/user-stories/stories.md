# 사용자 스토리 (User Stories)

**구성 방식**: Hybrid (페르소나 1차 그룹화 + 기능 2차 그룹화)
**세분화 수준**: 중간 (기능 단위 스토리 + 체크리스트 수용 기준)
**우선순위**: 전체 MVP 범위 → 모두 **Must**
**수용 기준 형식**: 체크리스트
**INVEST 준수**: 각 스토리는 Independent, Negotiable, Valuable, Estimable, Small, Testable 기준을 만족하도록 작성

> 추적성: 각 스토리는 `requirements.md`의 기능 요구사항(FR) ID와 연결됩니다.

# EPIC-C: 고객 주문 경험 (페르소나: 고객)

## US-C1: 테이블 태블릿 자동 로그인 및 세션 관리
**추적성**: FR-C1
**스토리**: 매장 고객으로서, 별도 로그인 절차 없이 테이블 태블릿에서 즉시 주문할 수 있도록, 태블릿이 자동으로 인증되기를 원한다.

**수용 기준**:
- [ ] 관리자가 1회 초기 설정(매장 식별자, 테이블 번호, 테이블 비밀번호)을 수행할 수 있다.
- [ ] 최초 로그인 성공 시 로그인 정보가 브라우저 로컬(localStorage)에 저장된다.
- [ ] 이후 태블릿을 열면 저장된 정보로 자동 로그인된다.
- [ ] 자동 로그인 성공 후 기본 화면(메뉴 화면)이 표시된다.
- [ ] 저장된 인증 정보가 유효하지 않으면 재설정 안내가 표시된다.

**페르소나**: 고객 (설정은 관리자 수행)

---

## US-C2: 메뉴 조회 및 탐색
**추적성**: FR-C2
**스토리**: 매장 고객으로서, 매장의 메뉴를 쉽게 탐색하고 선택할 수 있도록, 카테고리별로 정리된 메뉴 화면을 보기를 원한다.

**수용 기준**:
- [ ] 메뉴 화면이 기본 화면으로 항상 표시된다.
- [ ] 메뉴가 카테고리별로 분류되어 표시된다.
- [ ] 카테고리 간 빠른 이동이 가능하다.
- [ ] 각 메뉴는 메뉴명, 가격, 설명, 이미지를 표시한다.
- [ ] 메뉴는 카드 형태로 표시되며, 버튼은 터치 친화적 크기(최소 44x44px)이다.
- [ ] 명확한 시각적 계층 구조를 가진다.

**페르소나**: 고객

---

## US-C3: 장바구니 관리
**추적성**: FR-C3
**스토리**: 매장 고객으로서, 주문 전에 선택한 메뉴를 모아두고 수정할 수 있도록, 장바구니 기능을 사용하기를 원한다.

**수용 기준**:
- [ ] 메뉴를 장바구니에 추가/삭제할 수 있다.
- [ ] 장바구니 내 각 항목의 수량을 증가/감소할 수 있다.
- [ ] 총 금액이 실시간으로 계산되어 표시된다.
- [ ] 장바구니를 한 번에 비울 수 있다.
- [ ] 장바구니는 클라이언트 로컬에 저장되어 페이지 새로고침 시에도 유지된다.
- [ ] 서버로의 전송은 주문 확정 시에만 발생한다.

**페르소나**: 고객

---

## US-C4: 주문 생성
**추적성**: FR-C4
**스토리**: 매장 고객으로서, 장바구니에 담은 메뉴를 실제 주문으로 확정할 수 있도록, 주문 생성 기능을 사용하기를 원한다.

**수용 기준**:
- [ ] 주문 확정 전 주문 내역을 최종 확인할 수 있다.
- [ ] 주문 확정 버튼으로 주문을 제출할 수 있다.
- [ ] 주문 성공 시 주문 번호가 표시된다.
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다.
- [ ] 주문 성공 화면이 5초간 표시된 후 메뉴 화면으로 자동 리다이렉트된다.
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니는 유지된다.
- [ ] 전송되는 주문 정보에 매장 식별 정보, 테이블 식별 정보, 메뉴 목록(메뉴명/수량/단가), 총 금액, 세션 ID가 포함된다.

**페르소나**: 고객

---

## US-C5: 현재 세션 주문 내역 조회
**추적성**: FR-C5
**스토리**: 매장 고객으로서, 내가 주문한 내역과 진행 상태를 확인할 수 있도록, 현재 테이블 세션의 주문 내역을 조회하기를 원한다.

**수용 기준**:
- [ ] 주문 내역이 주문 시간 순으로 정렬되어 표시된다.
- [ ] 각 주문은 주문 번호, 주문 시각, 메뉴 및 수량, 주문 금액, 주문 상태(대기중/준비중/완료)를 표시한다.
- [ ] 현재 테이블 세션의 주문만 표시된다(이전 세션 및 이용 완료 주문 제외).
- [ ] 주문 상태가 SSE를 통해 실시간으로 갱신된다. (US-T1 참조)
- [ ] 주문이 많을 경우 페이지네이션 또는 무한 스크롤로 표시된다.

**페르소나**: 고객

# EPIC-A: 매장 운영 관리 (페르소나: 관리자)

## US-A1: 매장 인증 (로그인)
**추적성**: FR-A1
**스토리**: 매장 관리자로서, 내 매장 관리 시스템에 안전하게 접근할 수 있도록, 매장 인증(로그인) 기능을 사용하기를 원한다.

**수용 기준**:
- [ ] 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있다.
- [ ] 로그인 성공 시 JWT 토큰이 발급되고 16시간 동안 세션이 유지된다.
- [ ] 브라우저 새로고침 시에도 세션이 유지된다.
- [ ] 16시간 경과 시 자동 로그아웃된다.
- [ ] 비밀번호는 bcrypt로 해싱되어 저장된다.
- [ ] 반복적인 로그인 실패 시 로그인 시도가 제한된다.
- [ ] 잘못된 자격 증명에 대해 명확한 에러 메시지가 표시된다.

**페르소나**: 관리자 (세부 인증 메커니즘은 US-T2 참조)

---

## US-A2: 실시간 주문 모니터링
**추적성**: FR-A2
**스토리**: 매장 관리자로서, 들어오는 주문을 빠짐없이 관리할 수 있도록, 테이블별 주문 현황을 실시간 대시보드로 모니터링하기를 원한다.

**수용 기준**:
- [ ] 주문 목록이 SSE를 통해 실시간으로 업데이트된다(신규 주문 2초 이내 표시). (US-T1 참조)
- [ ] 대시보드가 테이블별 카드 형태의 그리드 레이아웃으로 표시된다.
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 n개 미리보기가 표시된다.
- [ ] 각 주문은 테이블 번호, 주문 번호, 주문 시각, 메뉴 및 수량(축약), 총 금액을 표시한다.
- [ ] 주문 카드 클릭 시 전체 메뉴 목록 상세를 볼 수 있다.
- [ ] 주문 상태를 대기중/준비중/완료로 변경할 수 있다.
- [ ] 신규 주문은 색상 변경/애니메이션으로 시각적으로 강조된다.
- [ ] 테이블별 필터링이 가능하다.

**페르소나**: 관리자

---

## US-A3: 테이블 관리 (세션 라이프사이클 및 주문 정정)
**추적성**: FR-A3
**스토리**: 매장 관리자로서, 테이블별 주문 상태와 세션을 관리할 수 있도록, 테이블 초기 설정·주문 삭제·이용 완료·과거 내역 조회 기능을 사용하기를 원한다.

**수용 기준 - 테이블 태블릿 초기 설정**:
- [ ] 테이블 번호와 테이블 비밀번호를 설정할 수 있다.
- [ ] 설정 시 16시간 세션이 생성되고 자동 로그인이 활성화된다.

**수용 기준 - 주문 삭제(직권 수정)**:
- [ ] 특정 주문에 대한 삭제 버튼이 제공된다.
- [ ] 삭제 전 확인 팝업이 표시된다.
- [ ] 확인 시 주문이 즉시 삭제된다.
- [ ] 삭제 후 테이블 총 주문액이 재계산된다.
- [ ] 성공/실패 피드백이 표시된다.

**수용 기준 - 테이블 세션 처리(이용 완료)**:
- [ ] 세션 시작(첫 주문)과 종료(이용 완료)를 관리할 수 있다.
- [ ] 이용 완료 처리 전 확인 팝업이 표시된다.
- [ ] 이용 완료 시 해당 세션의 주문 내역이 과거 이력으로 이동된다.
- [ ] 이용 완료 시 테이블의 현재 주문 목록 및 총 주문액이 0으로 리셋된다.
- [ ] 새 고객이 이전 주문 내역 없이 시작할 수 있다.
- [ ] 성공/실패 피드백이 표시된다.

**수용 기준 - 과거 주문 내역 조회**:
- [ ] "과거 내역" 버튼이 제공된다.
- [ ] 테이블별 과거 주문 목록이 시간 역순으로 표시된다.
- [ ] 각 과거 주문은 주문 번호, 시각, 메뉴 목록, 총 금액, 매장 이용 완료 시각을 표시한다.
- [ ] 날짜 필터링이 가능하다.
- [ ] "닫기" 버튼으로 대시보드로 복귀할 수 있다.

**페르소나**: 관리자

---

## US-A4: 메뉴 관리
**추적성**: FR-A4
**스토리**: 매장 관리자로서, 메뉴 정보를 자유롭게 관리할 수 있도록, 메뉴 등록·수정·삭제·순서 조정 기능을 사용하기를 원한다.

**수용 기준**:
- [ ] 카테고리별로 메뉴를 조회할 수 있다.
- [ ] 메뉴를 등록할 수 있다(메뉴명, 가격, 설명, 카테고리, 이미지 URL).
- [ ] 기존 메뉴를 수정할 수 있다.
- [ ] 메뉴를 삭제할 수 있다.
- [ ] 메뉴 노출 순서를 조정할 수 있다.
- [ ] 필수 필드 검증이 수행된다.
- [ ] 가격 범위 검증이 수행된다.

**페르소나**: 관리자

# EPIC-T: 기술 인에이블러 (교차 관심사, 페르소나: 공통)

> Lean-Hybrid 방식에 따라, 두 페르소나에 걸친 교차 관심사만 별도 기술 스토리로 분리합니다.

## US-T1: SSE 기반 실시간 업데이트 인프라
**추적성**: FR-A2, FR-C5, NFR-1, NFR-2
**스토리**: 시스템으로서, 관리자 대시보드와 고객 주문 내역에 주문 변경 사항을 실시간으로 전달할 수 있도록, Server-Sent Events(SSE) 기반 실시간 통신 인프라를 제공해야 한다.

**수용 기준**:
- [ ] 서버가 SSE 엔드포인트를 제공한다.
- [ ] 신규 주문 발생 시 관리자 대시보드에 2초 이내로 반영된다.
- [ ] 주문 상태 변경 시 해당 테이블의 고객 주문 내역 화면에 실시간 반영된다.
- [ ] 주문 삭제/세션 종료 등 변경 이벤트도 실시간으로 전달된다.
- [ ] 클라이언트 연결이 끊어진 경우 자동 재연결을 시도한다.

**페르소나**: 공통 (관리자 + 고객)

---

## US-T2: 인증 및 세션 인프라
**추적성**: FR-A1, FR-C1, NFR-3
**스토리**: 시스템으로서, 관리자와 테이블 태블릿의 인증을 일관되게 처리할 수 있도록, JWT 기반 인증 및 세션 관리 인프라를 제공해야 한다.

**수용 기준**:
- [ ] 관리자 인증은 JWT 토큰 기반으로 16시간 세션을 유지한다.
- [ ] 비밀번호(관리자, 테이블)는 bcrypt로 해싱되어 저장된다.
- [ ] 테이블 태블릿은 발급된 자격 증명으로 자동 로그인을 수행한다.
- [ ] 토큰 만료 시 적절히 거부되고 재인증이 요구된다.
- [ ] 인증이 필요한 API는 유효한 토큰 없이는 접근이 거부된다.

**페르소나**: 공통 (관리자 + 고객 태블릿)
